## How to Use This Changelog

Each version entry should include:
- **Added**: New features or content
- **Changed**: Modifications to existing content
- **Deprecated**: Features being phased out
- **Removed**: Deleted features or content
- **Fixed**: Bug fixes or corrections
- **Security**: Security-related changes

When making a new version:
1. Update this file with changes
2. Commit: `git commit -am "Version X.Y.Z: Brief summary"`
3. Tag: `git tag -a vX.Y.Z -m "Version X.Y.Z"`

## Version Numbering

- **Major (X.0.0)**: Significant restructuring or major new sections
- **Minor (1.X.0)**: New content, lexemes, or substantial additions
- **Patch (1.0.X)**: Bug fixes, typos, small refinements
